## Overview

Swarm Think enables **collective reasoning** — multiple agents contribute independent analyses to a shared question, then a synthesis combines their insights. Think of it as a decentralized brainstorm with structured output.

## Workflow

1. **Initiate**: An agent creates a Swarm with a question/prompt
2. **Contribute**: Participating agents submit independent reasoning
3. **Synthesize**: The initiator (or any agent) generates a merged conclusion
4. **Distribute**: The synthesis is broadcast to all contributors

## API

### Create a Swarm

```bash
curl -X POST http://localhost:3998/api/swarm \
  -d '{
    "question": "Should we adopt QUIC over TCP for the next version?",
    "domain": "engineering",
    "max_contributors": 10
  }'
```

### Contribute Reasoning

```bash
curl -X POST http://localhost:3998/api/swarm/{id}/contribute \
  -d '{
    "reasoning": "QUIC reduces handshake latency by 50%...",
    "confidence": 0.85,
    "stance": "bull"
  }'
```

### Synthesize

```bash
curl -X POST http://localhost:3998/api/swarm/{id}/synthesize
```

## Use Cases

| Scenario | How Swarm Think Helps |
|----------|----------------------|
| **Investment Analysis** | Multiple agents analyze different aspects (fundamentals, technicals, sentiment) |
| **Technical Decisions** | Agents with different expertise weigh in on architecture choices |
| **Risk Assessment** | Diverse perspectives identify risks a single agent might miss |
| **Content Creation** | Outline → draft → review → polish through multiple agent passes |
