The Scatter Problem

Current AI agents require context scattered across prompt files, loose requirements docs, and hidden API keys. It's permeable and messy.

Immutable Bundling

Nutshell creates a cryptographic seal around your context. Once packed, the `.nut` file guarantees the agent receives exactly what you intended, untampered.

Credential Injection

Securely embed temporary tokens or reference local credential stores without hardcoding secrets into plaintext prompts.

Standardized Schema

Stop inventing bespoke JSON formats. The `.nut` open standard is predictable, parseable, and natively understood by modern agent frameworks.

Anatomy of a Shell

A `.nut` file is a compressed archive containing layered, structured context. Just as a shell protects an organism, the archive encapsulates your requirements from external noise.

  • Manifest Envelope

    Defines metadata, target agent capabilities required, and token limits.

  • Context Strata

    The read-only workspace. Snapshots of code, documentation, and database schemas.

  • The Core Payload

    The specific, actionable directive. Protected, clear, and unambiguous.

Pack it up.

The Nutshell CLI makes bundling context as easy as creating a tarball, but with intelligence built for LLMs.

npm install -g @nutshell/cli

nut init auth-refactor
Initialized empty shell in ./auth-refactor.nut

nut add src/auth.ts tests/auth.test.ts
Added 2 files to Context Strata.

nut directive "Migrate to RS256"
Core payload updated.

nut pack --seal
✔ Shell sealed successfully: auth-refactor.nut (14kb)
Ready for transmission to Agent.
